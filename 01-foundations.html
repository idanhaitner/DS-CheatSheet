1.1 Asymptotic Notation

We describe the growth rate of a running-time function \(T(n)\), ignoring constants and low-order terms.

NotationMeaningFormal definitionAnalogy
O(g)Upper bound (≤)\(0 \le f(n) \le c\,g(n)\) for \(n \ge n_0\)\(f \le g\)
Ω(g)Lower bound (≥)\(0 \le c\,g(n) \le f(n)\) for \(n \ge n_0\)\(f \ge g\)
Θ(g)Tight bound (=)\(f = O(g)\) and \(f = \Omega(g)\)\(f = g\)
o(g)Strictly smaller (<)\(\forall c\,\exists n_0: f(n) < c\,g(n)\)\(f \ll g\)
ω(g)Strictly larger (>)\(\forall c\,\exists n_0: f(n) > c\,g(n)\)\(f \gg g\)
Growth hierarchy (memorize):

\(O(1) < O(\log\log n) < O(\log n) < O(\sqrt{n}) < O(n) < O(n\log n)\)

\(O(n^2) < O(n^3) < O(2^n) < O(n!) < O(n^n)\)

Key facts & tricks

  • Transitivity: if \(f=O(g)\) and \(g=O(h)\) then \(f=O(h)\).
  • Sum rule: \(O(f)+O(g)=O(\max(f,g))\) — the dominant term wins.
  • Logs: \(\log_a n = O(\log_b n)\) — base doesn't matter in \(O\).
  • Polynomial vs log: \(\log^k n = O(n^\epsilon)\) for any \(k,\epsilon>0\).
  • Polynomial vs exponential: \(n^k = O(c^n)\) for \(c>1\).
  • Limit test: \(\lim f/g = 0 \Rightarrow f=O(g)\); \(=\infty \Rightarrow g=O(f)\); constant \(\Rightarrow f=O(g)\) and \(g=O(f)\).
  • \(\log(n!) = O(n\log n)\) (Stirling) — gives the comparison-sort lower bound.
Pitfall: \(O\) is an upper bound, so "worst case is \(O(n^2)\)" and "worst case is \(O(n^3)\)" can both be true. If asked for a tight bound, use \(\Theta\).

1.2 Recurrences & the Master Theorem

Master Theorem

For \(T(n) = a\,T(n/b) + f(n)\) with \(a\ge 1,\ b>1\). Compare \(f(n)\) with the "watershed" \(n^{\log_b a}\):

CaseConditionSolution
1\(f(n) = O(n^{\log_b a - \epsilon})\) — \(f\) polynomially smaller\(T(n)=\Theta(n^{\log_b a})\)
2\(f(n) = \Theta(n^{\log_b a})\) — same order\(T(n)=\Theta(n^{\log_b a}\log n)\)
3\(f(n) = \Omega(n^{\log_b a + \epsilon})\), + regularity \(a f(n/b) \le c f(n),\ c<1\)\(T(n)=\Theta(f(n))\)
Gap trap: if \(f\) is larger only by a logarithmic factor (e.g. \(f=n^{\log_b a}\log n\)), the Master Theorem does not apply. Use the recursion-tree method.

Worked examples

Recurrence\(n^{\log_b a}\)CaseResult
Merge Sort: \(2T(n/2)+n\)\(n^1\)2\(\Theta(n\log n)\)
Binary Search: \(T(n/2)+1\)\(n^0\)2\(\Theta(\log n)\)
\(2T(n/2)+n^2\)\(n^1\)3\(\Theta(n^2)\)
\(4T(n/2)+n\)\(n^2\)1\(\Theta(n^2)\)
\(3T(n/4)+n\log n\)\(n^{0.79}\)3\(\Theta(n\log n)\)
\(T(n-1)+n\)—iterate\(\Theta(n^2)\)

Other methods

  • Substitution: guess the form, prove by induction (verify base case and inductive step separately).
  • Recursion tree: sum work per level × number of levels; great for guessing.
  • Iteration: unfold the recurrence to the base case, count levels.

1.3 Useful Sums & Identities

Arithmetic: \(\sum_{i=1}^{n} i = \dfrac{n(n+1)}{2} = \Theta(n^2)\)

Squares: \(\sum_{i=1}^{n} i^2 = \dfrac{n(n+1)(2n+1)}{6} = \Theta(n^3)\)

Geometric (\(x\ne1\)): \(\sum_{i=0}^{n} x^i = \dfrac{x^{n+1}-1}{x-1}\)

Convergent (\(|x|<1\)): \(\sum_{i=0}^{\infty} x^i = \dfrac{1}{1-x}\)

Harmonic: \(\sum_{i=1}^{n}\dfrac1i = \Theta(\log n)\)

Logs: \(\sum_{i=1}^{n}\log i = \log(n!) = \Theta(n\log n)\)

Powers of two: \(\sum_{i=0}^{k} 2^i = 2^{k+1}-1\)

Perfect binary tree of height \(h\): \(2^{h+1}-1\) nodes, \(2^h\) leaves.

1.4 Probability Basics

Randomized structures (skip lists, hashing, randomized quicksort/select) use expected-time analysis, averaged over the algorithm's coin flips — not over inputs.

ConceptDefinition / formula
Probability\(P(A)\in[0,1]\); \(P(\Omega)=1\); \(P(\emptyset)=0\)
Union\(P(A\cup B)=P(A)+P(B)-P(A\cap B)\)
Independence\(P(A\cap B)=P(A)\cdot P(B)\)
Conditional\(P(A\mid B)=\dfrac{P(A\cap B)}{P(B)}\)
Expectation\(E[X]=\sum_i x_i\,P(X=x_i)\)
Linearity\(E[X+Y]=E[X]+E[Y]\) — always, even if dependent
Indicator\(X_A=1\) if A occurs else 0; then \(E[X_A]=P(A)\)
The key exam technique — indicators + linearity: to count expected occurrences of many events, define an indicator \(X_i\) per event, then \(E[\text{total}]=\sum_i P(\text{event }i)\). This proves randomized quicksort/select bounds and skip-list height \(O(\log n)\).
  • Bernoulli(p): \(E[X]=p\). Trials until first success (Geometric): \(E=1/p\) (skip-list level: \(p=\tfrac12\Rightarrow\) expected 2 flips).
  • Uniform over m outcomes: each has probability \(1/m\) (basis of simple uniform hashing).
