6.1 Definitions & Representations

Graph \(G=(V,E)\): vertices + edges. Directed or undirected.

Degree — # incident edges.

Path — sequence of connected vertices; simple if no repeats.

Cycle — path returning to start.

Connected — path between every pair (undirected).

Connected component — maximal connected subgraph.

Tree — connected, acyclic, undirected (\(|E|=|V|-1\)).

DAG — directed acyclic graph.

6.2 BFS — Breadth-First Search

Idea: explore the graph in layers from source \(s\) — all vertices at distance 1, then distance 2, and so on. Uses a FIFO queue (first discovered, first explored).

What BFS is good for

ProblemWhy BFS?
Unweighted shortest path from \(s\)First time you reach \(v\), you used the fewest edges. \(dist[v]\) = hop count.
Reachability / connectivityVisits exactly the vertices reachable from \(s\). Run from each unvisited vertex to list all components.
Bipartite test (2-coloring)BFS levels alternate — if an edge connects same color, graph is not bipartite.
Level-order traversalSame wave idea as BFS on trees — process by depth from root.

When not to use BFS

  • Weighted shortest path → use Dijkstra (non-negative) or Bellman-Ford (negative OK).
  • Cycle detection in directed graphs → prefer DFS (back edge is easier to spot).
  • Topological order → DFS on a DAG, not BFS.
Exam mnemonic: “shortest path, all edges cost 1” → BFS. Complexity always O(V + E) with adjacency list — each vertex and edge touched once.

BFS(G, s):
    color all white, dist = infinity
    dist[s] = 0
    enqueue(s)
    while Q not empty:
        u = dequeue()
        for v in Adj[u]:
            if color[v] == white:
                dist[v]   = dist[u] + 1
                parent[v] = u
                enqueue(v)

Time O(V + E) · Space O(V) (queue + color/dist arrays).

BFS tree: edges used when first discovering a white vertex form a tree. Not every shortest path is a tree edge — but tree distance equals shortest distance. Exams sometimes ask how many distinct BFS trees exist (depends on tie-breaking when enqueueing neighbors).

6.3 DFS — Depth-First Search

Idea: from a vertex, go as deep as possible along one path, then backtrack. Uses a stack (explicit) or recursion (call stack). Records discovery \(d[v]\) and finish \(f[v]\) timestamps.

What DFS is good for

ProblemWhy DFS?
Cycle detection (directed)A back edge to a gray (active) ancestor means a cycle exists.
Topological sortOn a DAG: output vertices in decreasing finish time (or prepend when finished).
Connected componentsEach DFS tree in the forest = one component (undirected).
Strongly connected componentsTwo-pass DFS (Kosaraju) or Tarjan — advanced, same \(O(V+E)\) spirit.
Path existence / mazeExplores all reachable vertices; does not guarantee shortest path.
Parenthesis theorem / intervalsNested \( [d[v], f[v]] \) intervals classify edge types — common proof question.

Edge types (directed graph)

Edge (u → v)Meaning
Tree\(v\) white when explored from \(u\) — discovery edge in DFS forest.
Back\(v\) is gray (ancestor of \(u\)) — cycle in directed graph.
Forward\(v\) black, \(v\) descendant of \(u\), not a tree edge.
Cross\(v\) black, not ancestor/descendant — between different branches.

In an undirected graph DFS only has tree and back edges (back = to an already discovered ancestor).

Exam mnemonic: “cycles, topo sort, finish times, components” → DFS. Same O(V + E) time as BFS with adjacency list.
white → gray → black tree edge back edge rejected

pseudocode 
DFS-Visit(u):
    color[u] = gray
    d[u] = ++time
    for v in Adj[u]:
        if color[v] == white:
            parent[v] = u
            DFS-Visit(v)
    color[u] = black
    f[u] = ++time

Time O(V + E) · Space O(V) (recursion/stack depth up to \(O(V)\)).

Cycle detection: in a directed graph, a back edge (to a gray node) ⟺ cycle. In undirected graphs, back edge to parent’s ancestor also indicates a cycle.
White-path theorem: \(v\) is a descendant of \(u\) in the DFS forest ⟺ at time \(d[u]\) there is an all-white path \(u\rightsquigarrow v\).

6.5 Minimum Spanning Tree (MST)

For a connected, weighted, undirected graph: a spanning tree (\(|V|-1\) edges) of minimum total weight. Both algorithms are greedy.

white → gray → black MST edge back edge rejected (cycle)

Step 0 / 0

Kruskal

  1. Sort all edges by weight ascending.
  2. Add the next edge if it connects two different components (no cycle) — checked with Union-Find.
  3. Stop after \(|V|-1\) edges.

Time O(E log E) = O(E log V). Edge-centric; great for sparse graphs.

Prim

  1. Grow one tree from an arbitrary root.
  2. Each vertex has key = min edge weight to the tree.
  3. Repeatedly extract-min vertex, add it, update neighbor key values.

Time O(E log V) (binary heap).

Cut property: for any cut, the min-weight crossing edge is safe to add. Cycle property: the max-weight edge in any cycle is never in an MST.

6.6 Dijkstra — Single-Source Shortest Paths

Shortest paths from \(s\) with non-negative weights. Greedy: repeatedly finalize the closest unvisited vertex, then relax each outgoing edge \((u,v)\) — if \(dist[u]+w(u,v) < dist[v]\), update \(dist[v]\).

pseudocode 
Dijkstra(G, w, s):
    dist[s] = 0, others = infinity
    PQ = all vertices
    while PQ not empty:
        u = extractMin(PQ)
        for v in Adj[u]:
            if dist[u] + w(u,v) < dist[v]:
                dist[v] = dist[u] + w(u,v)
                decreaseKey(v)
Binary heapO((V+E) log V)
Fibonacci heapO(E + V log V)
Fails with negative edges! Once a vertex is finalized it's never revisited. Use Bellman-Ford (O(VE)) for negative weights.
BFS vs Dijkstra: BFS = Dijkstra with all weights 1. On unweighted graphs, prefer BFS.
