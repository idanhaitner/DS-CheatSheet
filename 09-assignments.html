9 · Homework Assignments 2 & 3

Exam-oriented summary of the course HWs. Past exams often reuse these design patterns.

Why this page? Design-a-structure questions on the exam often mirror HW ideas: combine two structures, keep local aux fields, lazy updates, IndexableSkipList width/span, hash + ordered structure. Link into the theory pages when you need depth.

Assignment 2 — Lists, AVL, composite ADTs

Theme: implement primitives, then design composite structures (only given classes + arrays + primitives — no Java collections).

A. Building blocks

TaskTimeIdea
MyArray.reverse / MyLinkedList.reverseO(n)In-place reverse (swap ends / swap prev–next + head–tail).
MyAVLTree.depthOfMinO(log n)Walk left from root; count edges (empty → −1).

Toolkit you already have: unsorted / sorted array, unsorted / sorted DLL, full AVL. Know which ops are \(O(1)\) with a pointer (DLL delete, successor).

B. MyDynamicSet — classic dynamic set

Implemented with a sorted doubly linked list.

OpTime
Search / InsertO(n)
Delete (node pointer)O(1)
Min / Max / Succ / PredO(1)
This is the “naive ordered set.” On the exam, if you need \(\log n\) Search/Insert, upgrade to AVL / skip list / OST — not a sorted list.

C. MyFirstDataStructure — AVL + list

Need: fast by-key ops and insertion-order first/last and maximum key.

OpTimeHow
InsertO(log n)AVL insert + list insert-at-head \(O(1)\); update cached max.
FindAndRemove(k)O(log n)AVL search → unlink list node via cross-pointer \(O(1)\) → AVL delete; refresh max if needed.
MaximumO(1)Cached maxElement.
First / LastO(1)List head = newest, tail = oldest (insert at head).
Exam pattern: cross-pointers between AVL node and list node. Delete by key stays \(O(\log n)\) without scanning the list. Same idea as hash storing a skip-list node pointer in HW3.

D. MySecondDataStructure — products + stats

Products: id, quality \(\in\{0,\ldots,5\}\), price. Array by id + aggregates.

OpTimeHow
InsertO(1)Array insert; bump qualityCounter[q], qualitiesSum, per-quality max.
FindAndRemove(id)O(n)Linear search + swap-delete; rebuild that quality’s max if needed.
MedianQualityO(1)Walk 6 counters to index \((n+1)/2\).
AvgQualityO(1)qualitiesSum / n.
RaisePrice(raise, q)O(1)Lazy qualityPriceAddition[q] += raise — do not touch all products.
MostExpensiveO(1)Max over ≤6 candidates using price + addition[q].
Exam patterns: (1) small discrete domain → histogram for median/avg in \(O(1)\). (2) lazy additive offsets instead of updating \(n\) items. (3) cache per-bucket extrema.

Assignment 3 — Hashing + Indexable Skip List

Theme: universal hashing, chaining vs probing, experiments, indexable skip list (spans), then combine hash ⊕ skip list.

A. Hash families & tables

PieceWhat to know
Modular hash\(h(k)=((ak+b)\bmod p)\bmod m\), often \(m=2^k\).
Multiplicative shifting\((a\cdot key) \ggg (64-k)\).
ChainingInsert / search / delete; rehash when \(\alpha >\) max (double capacity).
ProbingLinear probing + tombstones on delete; same rehash idea.

Expected dictionary ops O(1) under good \(\alpha\); one rehash O(n) (amortized \(O(1)\) per insert with doubling). See §5.2.

Experiments: measure insert/search time vs \(\alpha\) for probing \(\{0.5,0.75,7/8,15/16\}\) and chaining \(\{0.5,0.75,1,1.5,2\}\).

B. IndexableSkipList (spans / width)

Skip list + per-level span (= width): # of bottom-level nodes spanned by that forward jump. Same idea as §5.1.1.

OpExpectedIdea
Find / Insert / DeleteO(log n)Standard skip path; with spans, Delete updates the whole search path (not \(O(1)\)).
Rank(key)O(log n)Sum spans while searching → # of keys \(<\) key.
Select(i)O(log n)Jump by span; inverse of Rank (0-based index in HW).
Min / Max / Pred / SuccO(1)Sentinels + L0 links.
Never store absolute index on each node. Use local spans/widths (or OST size). See order-stat toolkit.

C. MyDataStructure — hash ⊕ indexable skip list

Need: membership \(O(1)\) expected and Rank / Select / range in expected \(O(\log n)\) / output-sensitive.

OpExpectedHow
Init(N)WC Θ(N)Chained table capacity \(\Theta(N)\) + empty skip list.
ContainsΘ(1)Hash only.
InsertΘ(log n)Hash duplicate check → skip insert → store node pointer in hash.
Deletemiss Θ(1) · hit Θ(log n)Hash lookup → skip delete by pointer → hash delete.
Rank / SelectΘ(log n)Delegated to skip list spans.
Range(low, high)Θ(|L|)If low present, walk L0 while \(\le\) high.
Exam pattern: Hash for membership + ordered structure for order stats. Storing the skip/AVL node pointer in the hash is what makes Delete after lookup fast.
