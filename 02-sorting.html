Comparison sorts (insertion, selection, bubble, merge, quick, heap) cannot beat Ω(n log n) worst case. Non-comparison sorts (counting, radix, bucket) break this by assuming structure on the input.

Insertion Sort O(n²)

Idea. Grow a sorted prefix on the left. Take the next "key" and shift larger sorted elements right until the key lands in place — like sorting cards in your hand.

pseudocode 
for j = 1 to n-1:
    key = A[j]
    i   = j - 1
    while i >= 0 and A[i] > key:
        A[i+1] = A[i]        # shift right
        i = i - 1
    A[i+1] = key             # drop key into the gap
Best (already sorted)Θ(n)
Average / Worst (reverse)Θ(n²)
SpaceO(1), in-place, stable
Exam gold: runs in O(n·d) where d = number of inversions, or O(nk) when every element is ≤ k positions from its final spot. Best for nearly-sorted or small arrays.

Selection Sort O(n²)

Idea. Repeatedly find the minimum of the unsorted suffix and swap it to the front. Always \(\Theta(n^2)\) even if already sorted, but does only n−1 swaps.

pseudocode 
for i = 0 to n-2:
    min = i
    for j = i+1 to n-1:
        if A[j] < A[min]:
            min = j
    swap(A[i], A[min])
Useful when writes are expensive (only n−1 swaps total). Not stable.

Bubble Sort O(n²)

Idea. Repeatedly walk the array swapping adjacent out-of-order pairs; the largest element "bubbles" to the end each pass. With an early-exit flag it is \(\Theta(n)\) on sorted input.

pseudocode 
for i = 0 to n-2:
    swapped = false
    for j = 0 to n-2-i:
        if A[j] > A[j+1]:
            swap(A[j], A[j+1])
            swapped = true
    if not swapped:
        break                # already sorted, stop early

Best Θ(n) · Average/Worst Θ(n²) · O(1) space, stable.

Merge Sort O(n log n)

Idea. Divide the array in half, recursively sort each half, then merge the two sorted halves in linear time. \(T(n)=2T(n/2)+\Theta(n)\Rightarrow \Theta(n\log n)\) in all cases.

pseudocode 
MergeSort(A, lo, hi):
    if lo < hi:
        mid = (lo + hi) / 2
        MergeSort(A, lo, mid)
        MergeSort(A, mid+1, hi)
        Merge(A, lo, mid, hi)     # O(n): two pointers, copy smaller

Quick Sort avg n log n worst n²

Idea. Pick a pivot, partition so smaller elements go left and larger go right (pivot lands in its final spot), then recurse on each side. In-place, small constants.

partition (Lomuto, pivot = last) 
Partition(A, lo, hi):
    pivot = A[hi]
    i = lo - 1
    for j = lo to hi-1:
        if A[j] <= pivot:
            i = i + 1
            swap(A[i], A[j])
    swap(A[i+1], A[hi])          # put pivot in place
    return i+1
Best / AverageΘ(n log n)
Worst (sorted + last-pivot)Θ(n²)
Worst case occurs when the pivot is always min/max → splits of size (n−1, 0): \(T(n)=T(n-1)+\Theta(n)=\Theta(n^2)\). A deterministic last-element pivot triggers this on already-sorted input. Fix with a random or median-of-three pivot.

Heap Sort O(n log n)

Idea. Build a max-heap in O(n). Then repeatedly swap the root (max) with the last element, shrink the heap, and sift the new root down. In-place, guaranteed \(\Theta(n\log n)\).

pseudocode 
HeapSort(A):
    BuildMaxHeap(A)              # O(n)
    for i = n-1 downto 1:
        swap(A[0], A[i])         # move current max to the end
        heapSize = heapSize - 1
        MaxHeapify(A, 0)         # O(log n)
Advantage over merge sort: O(1) extra memory. Advantage over quicksort: guaranteed O(n log n). See Heaps for the heapify diagram.

Counting Sort O(n + k)

Idea (non-comparison). For integer keys in \([0,k]\): count occurrences, take prefix sums to know final positions, then place elements right to left (preserves stability).

pseudocode 
CountingSort(A, k):
    C = new int[k+1] filled with 0
    for a in A:          C[a] = C[a] + 1        # counts
    for i = 1 to k:      C[i] = C[i] + C[i-1]   # prefix sums
    for j = n-1 downto 0:                       # right to left = stable
        C[A[j]] = C[A[j]] - 1
        B[C[A[j]]] = A[j]

Bucket Sort avg O(n)

Idea. For inputs uniform over \([0,1)\): split into n equal buckets, drop each element into bucket \(\lfloor n\cdot x\rfloor\), sort each bucket (insertion sort), then concatenate.

pseudocode 
BucketSort(A):
    create n empty lists B[0..n-1]
    for x in A:
        append x to B[floor(n * x)]
    for i = 0 to n-1:
        insertionSort(B[i])
    return concat(B[0..n-1])

Average Θ(n) (uniform input) · Worst Θ(n²) (all in one bucket).

The Comparison-Sort Lower Bound

Any comparison-only sort is a decision tree: internal nodes are comparisons, leaves are permutations. There are \(n!\) orderings, so \(\ge n!\) leaves. A binary tree with \(L\) leaves has height \(\ge \log_2 L\), so worst-case comparisons \(\ge \log_2(n!) = \Theta(n\log n)\).

Conclusion: Merge/Heap sort are asymptotically optimal comparison sorts. Counting/Radix/Bucket beat this only by not being comparison-based.

Selection & Median (order statistics)

Find the \(i\)-th smallest element (the median is \(i=\lfloor n/2\rfloor\)). Sorting takes \(O(n\log n)\), but selection can be done in linear time. Use the visualizer above to watch Quickselect and Median-of-Medians.

Randomized Select (Quickselect) — expected O(n)

Like quicksort but recurse into only one side.

pseudocode 
RandSelect(A, lo, hi, i):
    if lo == hi:
        return A[lo]
    q = RandPartition(A, lo, hi)
    k = q - lo + 1               # rank of pivot within range
    if   i == k: return A[q]
    elif i < k:  return RandSelect(A, lo,  q-1, i)
    else:       return RandSelect(A, q+1, hi,  i-k)

Expected O(n) from \(T(n)\le T(3n/4)+O(n)\); worst O(n²).

Deterministic Select — median of medians — worst-case O(n)

  1. Divide into groups of 5; find each group's median (O(n) total).
  2. Recursively find the median \(x\) of those \(\lceil n/5\rceil\) medians.
  3. Partition around \(x\); recurse into the correct side.

\(T(n) \le T(n/5) + T(7n/10) + O(n) \Rightarrow T(n)=O(n)\)   (since \(\tfrac15+\tfrac{7}{10}<1\))

Why groups of 5? Odd size gives a clean median and keeps the two recursion fractions summing below 1. Groups of 3 give \(T(n/3)+T(2n/3)+O(n)=O(n\log n)\) — fails.
