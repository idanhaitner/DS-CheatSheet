5.1 Skip List

A randomized dictionary: stacked sorted linked lists. The bottom list has all elements; each higher list keeps each element with probability ½, giving \(\Theta(\log n)\) levels with high probability. Search drops down and moves right, skipping big gaps.

Expected bounds are over the random coin flips (not over input order). Worst-case search O(n) happens if every node stays on the bottom level only: the structure becomes a sorted linked list. Yes, the list grows forever as you insert, but we always measure cost in terms of the current n.
Not unbounded per operation: worst case is O(n), not infinite. What grows without bound is n itself. Real implementations often cap tower height at about O(log n).

Expectation analysis

Coin: each node keeps promoting until it stops. Write \(q\) = Prob(promote to next level), \(r=1-q\) = Prob(stop). Lecture default: \(q=\tfrac12\). Course Geometric = trials until first success, \(E[X]=1/p_{\mathrm{success}}\); height / steps often use \(X-1\).

Height vs right-moves: height stops on first stop flip (prob \(r\)) → \(E[H]=1/r-1\). Reverse-path right-moves stop on a promote (prob \(q\)) → \(E[R_i]\le 1/q-1\). Same number when \(q=\tfrac12\).

5.1.1 Indexable Skip List (order statistics)

Skip list extended for Select (and Rank): expected \(O(\log n)\) Insert / Delete / Search / Select. Extra field per level: width (“step width”).

Extra fields

Each node already has next[] / prev[]. Add width[ℓ] = number of bottom-level (level 0) nodes spanned by the skip from this node to next[ℓ] (including the landing node). Level 0 always has width = 1 for a real successor.

Operations (exam sketch)

OpIdeaTime
InitEmpty skip list (sentinels).O(1)
SearchOrdinary skip-list search.expected O(log n)
InsertOrdinary insert, then fix width along the search path: reverse from the new node up to height of the tower (local arithmetic), then for levels above the new height add \(+1\) to the relevant widths on the path.expected O(log n)
DeleteOrdinary unlink + width updates symmetric to Insert (must touch the whole search path, not only the deleted tower).expected O(log n)
Select(i)Like search, but compare against ranks: start pos = 0; at level \(ℓ\), if pos + width[ℓ] < i step right; if == i take that jump and return (exact hit); if greater, drop down.expected O(log n)
Rank(x)Like search by key: jump right while next.key < x, adding width each jump. Then Rank = pos + 1 if x is present (1-based).expected O(log n)
Pitfall: plain skip-list Delete with a pointer is \(\Theta(1)\) expected. With width, Delete is expected \(O(\log n)\) — update widths on every level of the search path, including levels above the deleted/inserted tower.
When to prefer this vs OST: see §2.4 toolkit. Worst-case → OST; expected OK → IndexableSkipList.

5.2 Hash Tables: the Dictionary ADT

Maps keys to array slots via a hash function \(h:U\to\{0,\ldots,m-1\}\). Load factor \(\alpha=n/m\) (items ÷ table size). Under simple uniform hashing, both strategies are fast on average when \(\alpha\) stays reasonable: but worst case is always O(n) if every key collides.

Load factor \(\alpha=n/m\). Chaining can use any \(\alpha\). Probing needs \(\alpha<1\) (empty slots required).

Expected = simple uniform hashing. Worst = every key collides. Probing averages are in probes (each probe is O(1)). Delete in chaining is Θ(1+α) without a node pointer (must search the chain first). Rehash: a single rehash costs O(n), but with doubling strategy it is O(1) amortized per insert (same idea as dynamic array).

Hash functions

  • Division: \(h(k)=k \bmod m\). Pick \(m\) a prime not near a power of 2.
  • Multiplication: \(h(k)=\lfloor m\,(kA\bmod 1)\rfloor\), \(A\approx(\sqrt5-1)/2\). \(m\) can be a power of 2.

5.3 Open Addressing (Probing)

All items live in the table (no lists). Requires \(\alpha<1\). On collision, probe until an empty (or tombstone) slot is found.

SchemeProbe \(h(k,i)\)Issue
Linear\((h'(k)+i)\bmod m\)Primary clustering
Quadratic\((h'(k)+c_1 i+c_2 i^2)\bmod m\)Secondary clustering; fewer probes
Double hashing\((h_1(k)+i\,h_2(k))\bmod m\)Best spread; \(h_2\) coprime with \(m\)

OperationExpectedWorst
InsertO(1) probesO(n)

Exact probe bounds (exam detail): unsuccessful ≤ \(\frac{1}{1-\alpha}\); successful ≤ \(\frac{1}{\alpha}\ln\frac{1}{1-\alpha}\). Delete probes the chain like search, then marks a tombstone in O(1). Rehash costs O(n) per event; with doubling, O(1) amortized per insert.

Deletion is tricky: you can't just empty a slot (breaks probe chains). Use a "deleted" tombstone marker. Tombstones raise probe lengths over time: rehash periodically if many deletes.

5.4 Bloom Filter

Space-efficient probabilistic membership test. Bit-array of size \(m\) + \(k\) hash functions. Insert: set bits \(h_1(x)\ldots h_k(x)\). Query: all k bits 1 → "probably present"; any bit 0 → definitely absent.

  • No false negatives, but possible false positives.
  • False-positive rate after n inserts ≈ \(\left(1-e^{-kn/m}\right)^k\).
  • Cannot delete. Query/insert are O(k) = O(1).
