5.1 Skip List

A randomized dictionary: stacked sorted linked lists. The bottom list has all elements; each higher list keeps each element with probability ½, giving \(\Theta(\log n)\) levels with high probability. Search drops down and moves right, skipping big gaps.

Expectation analysis (from lecture)

Coin model: each node starts at height 0; while the flip is Tails, promote one level; stop on Heads. So promotion probability to the next level is \(1/2\).

Distributions you need
NameMeaningExpectation
Bernoulli(\(p\)) One coin flip: success with prob. \(p\) \(E[X]=p\)
Geometric(\(p\)) Number of trials until first success (incl. the success) \(E[X]=1/p\). With \(p=\tfrac12\): \(E=2\)
Binomial(\(n,p\)) Number of successes in \(n\) independent Bernoulli(\(p\)) trials \(E[X]=np\)

Some texts define Geometric as “number of failures before first success” (then \(E=(1-p)/p\)). In this course: height of a node \(+1\) is Geometric with \(p=\tfrac12\), so \(E[\text{height}+1]=2\) and \(E[\text{height}]=1\).

1) Expected height of one node

Insert flips until Heads. Let \(H\) = height of the node. Then \(H+1\sim\mathrm{Geometric}(1/2)\):

\[ E[H+1]=2 \quad\Rightarrow\quad E[H]=1. \]

That is why Delete (given a pointer) is \(\Theta(1)\) expected: you unlink from \(H+1\) lists, and \(E[H+1]=2\).

2) Expected size of the whole structure (Binomial + linearity)

An element is in list \(S_i\) iff its first \(i\) flips were Tails → probability \(1/2^i\).

\(|S_i|\) (not counting sentinels) is Binomial(\(n, 1/2^i\)), so \(E[|S_i|]=n/2^i\).

\[ E\!\left[\sum_{i=0}^{\infty}|S_i|\right] =\sum_{i=0}^{\infty} E[|S_i|] =\sum_{i=0}^{\infty}\frac{n}{2^i} = n\cdot 2 = 2n. \]

With sentinels: expected total size is \(2n + O(\log n)\).

3) Expected height of the skip list

Let \(X_i\) be the indicator “\(S_i\) is non-empty”. Height \(h=\sum_{i\ge 1} X_i\). Then \(E[X_i]\le\min(1, n/2^i)\). Split at \(\lfloor\log n\rfloor\):

\[ E[h] \le \sum_{i=1}^{\lfloor\log n\rfloor} 1 + \sum_{i=\lfloor\log n\rfloor+1}^{\infty}\frac{n}{2^i} \le \log n + 2. \]
4) Expected number of rightward steps on a certain level

Goal: for one fixed level \(S_i\), bound \(E[R_i]\), where \(R_i\) = number of rightward steps Search makes on that level.

Reverse-path trick (lecture): look at the Find path backwards. Start at the answer node in \(S_0\). Prefer going up; if you cannot go up, go left. This reverse path visits the same nodes as Search, just in reverse order.

On one level \(S_i\), the reverse path walks left across nodes whose coin for “promote further” was Heads (they stop at this height), until it hits a node that can go up (that node’s relevant flip was Tails), or hits the sentinel.

So the walk on this level is like a Geometric experiment with success probability \(1/2\):

  • Let \(X\sim\mathrm{Geometric}(1/2)\) = number of flips until first Heads (\(E[X]=2\)).
  • The lecture counts only the leftward/rightward moves, not the last “success” flip that lets you leave the level.
  • Also, the walk may stop early at the sentinel.

Therefore \(R_i \le X - 1\), which gives the main bound for a certain level:

\[ E[R_i]\le E[X]-1 = 2-1 = 1. \]

In words: on any single level, the expected number of rightward steps is at most 1.

Second bound (useful for high levels that are almost empty): \(R_i\le|S_i|\), so

\[ E[R_i]\le E[|S_i|] = \frac{n}{2^i}. \]

Use \(E[R_i]\le 1\) for levels \(i\le\log n\), and \(E[R_i]\le n/2^i\) for higher levels.

5) Total Search cost (sum over levels)

Total path length \(R = h + \sum_{i=0}^{\infty} R_i\) (ups between levels + right moves on each level):

\[ E[R] = E[h] + \sum_{i=0}^{\infty} E[R_i] \le E[h] + \sum_{i=0}^{\lfloor\log n\rfloor} 1 + \sum_{i=\lfloor\log n\rfloor+1}^{\infty}\frac{n}{2^i} \le 2\log n + 5. \]

Hence Search / Insert are \(\Theta(\log n)\) expected.

Exam takeaway: Geometric → one node’s height, and \(E[R_i]\le 1\) right moves per level; Binomial → how many nodes sit on level \(i\); indicators + linearity → height of the whole list and total search cost.

5.2 Hash Tables: the Dictionary ADT

Maps keys to array slots via a hash function \(h:U\to\{0,\ldots,m-1\}\). Load factor \(\alpha=n/m\) (items ÷ table size). Under simple uniform hashing, both strategies are fast on average when \(\alpha\) stays reasonable: but worst case is always O(n) if every key collides.

Load factor \(\alpha=n/m\). Chaining can use any \(\alpha\). Probing needs \(\alpha<1\) (empty slots required).

Expected = simple uniform hashing. Worst = every key collides. Probing averages are in probes (each probe is O(1)). Delete in chaining is Θ(1+α) without a node pointer (must search the chain first). Rehash: a single rehash costs O(n), but with doubling strategy it is O(1) amortized per insert (same idea as dynamic array).

Hash functions

  • Division: \(h(k)=k \bmod m\). Pick \(m\) a prime not near a power of 2.
  • Multiplication: \(h(k)=\lfloor m\,(kA\bmod 1)\rfloor\), \(A\approx(\sqrt5-1)/2\). \(m\) can be a power of 2.

5.3 Open Addressing (Probing)

All items live in the table (no lists). Requires \(\alpha<1\). On collision, probe until an empty (or tombstone) slot is found.

SchemeProbe \(h(k,i)\)Issue
Linear\((h'(k)+i)\bmod m\)Primary clustering
Quadratic\((h'(k)+c_1 i+c_2 i^2)\bmod m\)Secondary clustering; fewer probes
Double hashing\((h_1(k)+i\,h_2(k))\bmod m\)Best spread; \(h_2\) coprime with \(m\)

Operation costs (probing)

OperationExpectedWorst
InsertO(1) probesO(n)

Rehash (table doubles)O(n) one-time: re-insert all n keys into new table of size 2m

Exact probe bounds (exam detail): unsuccessful ≤ \(\frac{1}{1-\alpha}\); successful ≤ \(\frac{1}{\alpha}\ln\frac{1}{1-\alpha}\). Delete probes the chain like search, then marks a tombstone in O(1). Rehash costs O(n) per event; with doubling, O(1) amortized per insert.

Deletion is tricky: you can't just empty a slot (breaks probe chains). Use a "deleted" tombstone marker. Tombstones raise probe lengths over time: rehash periodically if many deletes.

5.4 Bloom Filter

Space-efficient probabilistic membership test. Bit-array of size \(m\) + \(k\) hash functions. Insert: set bits \(h_1(x)\ldots h_k(x)\). Query: all k bits 1 → "probably present"; any bit 0 → definitely absent.

  • No false negatives, but possible false positives.
  • False-positive rate after n inserts ≈ \(\left(1-e^{-kn/m}\right)^k\).
  • Cannot delete. Query/insert are O(k) = O(1).
