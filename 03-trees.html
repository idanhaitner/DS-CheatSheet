3.1 Stack & Queue

Stack: LIFO (Last In, First Out)

push →
[ a b c ]
← pop (c)

All ops O(1): push, pop, peek, isEmpty.

Uses: call stack, DFS, expression evaluation, backtracking, undo.

Queue: FIFO (First In, First Out)

enqueue →
[ a b c ]
→ dequeue (a)

All ops O(1): enqueue, dequeue, front, isEmpty.

Uses: BFS, scheduling, buffering. A deque gives O(1) at both ends.

3.3 Tree Definitions & Traversals

Root: node with no parent.

Leaf: node with no children.

Depth: # edges from root to node.

Height: # edges on longest path to a leaf below.

Tree height: height of root (empty = −1).

Degree: number of children.

Binary tree: each node ≤ 2 children.

Full: every internal node has exactly 2 children.

Perfect: full + all leaves same depth (\(2^{h+1}-1\) nodes).

Complete: all levels full except last, filled left→right (heaps).

Traversals: all O(n)

TraversalOrderOn a BST gives
In-orderLeft → Root → Rightsorted order
Pre-orderRoot → Left → Rightcopy / serialize
Post-orderLeft → Right → Rootdelete tree, compute heights

3.4 Binary Search Tree (BST)

BST property: for every node, all left-subtree keys are < node, all right-subtree keys are > node. Operations follow one root-to-leaf path → O(h).

In-order → 1, 3, 6, 8, 10, 14 (sorted)
OperationTime
Search / Insert / DeleteO(h)
Min / MaxO(h)
Successor / PredecessorO(h)
In-order traversalO(n)
h ranges from log n (balanced) to n (a "stick"). Inserting sorted data into a plain BST builds a stick → O(n) per op. Hence AVL.

Successor / Predecessor

  • Successor: if right child exists → min of right subtree; else → lowest ancestor whose left child is also an ancestor of x.
  • Predecessor: if left child exists → max of left subtree; else → lowest ancestor whose right child is also an ancestor of x.

Deletion: 3 cases

  1. No children: remove it.
  2. One child: splice it out (connect parent to child).
  3. Two children: replace with successor (min of right subtree, has no left child), then delete the successor.

3.5 AVL Tree: self-balancing BST

Invariant: for every node, \(|h_{left}-h_{right}|\le 1\). Forces height \(h=\Theta(\log n)\) → all ops O(log n) worst case. Each node stores its height.

OperationTime
SearchO(log n)
InsertO(log n): at most 1 (single/double) rotation
DeleteO(log n): up to O(log n) rotations to the root

The 4 rotation cases

CaseShapeFix
LLleft subtree of left childsingle Right-Rotate
RRright subtree of right childsingle Left-Rotate
LRright subtree of left childLeft-Rotate(child) then Right-Rotate
RLleft subtree of right childRight-Rotate(child) then Left-Rotate

Left-Rotation (fixes RR)

Before: right-heavy (RR)
After Left-Rotate(z): balanced
Mnemonic: the rotation is the opposite of the heavy side. Right-heavy → rotate left; left-heavy → rotate right. Double cases first rotate the child to reduce to a single case.

3.6 Order Statistic Tree

An order statistic tree is an AVL (or any balanced BST) extended with a size field on every node. It implements a Dynamic Ordered Set: all usual ordered-set ops, plus Select and Rank, each in \(\Theta(\log n)\).

New operations

OperationMeaningTime
Select(S, i)Return the \(i\)-th smallest key (\(i=1\) is the minimum)\(\Theta(\log n)\)
Rank(S, k)Return the rank of key \(k\) in \(S\) (\(1\) = smallest)\(\Theta(\log n)\)
Example: \(S=\{2,9,3,8,20,1,5\}\) sorted is \(1,2,3,5,8,9,20\).
\(\mathrm{Select}(S,1)=1\), \(\mathrm{Select}(S,3)=3\), \(\mathrm{Rank}(S,5)=4\), \(\mathrm{Rank}(S,20)=7\).

The extra field: size

For every node \(u\):

\[ u.\mathit{size} = 1 + u.\mathit{left}.\mathit{size} + u.\mathit{right}.\mathit{size} \]

(missing child ⇒ size \(0\)). A leaf has \(\mathit{size}=1\). Extra memory \(\Theta(n)\).

Why this helps: in a BST, keys smaller than a node sit in its left subtree, and also in left subtrees of ancestors when the search path went right. With size, you can count those keys in \(O(1)\) per node along a root-to-leaf path.

Maintaining size on insert / delete

  • Insert a leaf with \(\mathit{size}=1\).
  • After insert/delete/rotations, recompute \(\mathit{size}\) bottom-up on the affected path (and rotated nodes).
  • The formula is local and \(O(1)\), so insert/delete stay \(\Theta(\log n)\).

Search / Min / Max / Succ / Pred stay as in AVL: they ignore size.

3.7 B-Tree (minimum degree t)

  

    
Balanced search tree optimized for disk (minimize block reads). Each node holds many keys, so the tree is very shallow. Every node except root has between t−1 and 2t−1 keys; an internal node with k keys has k+1 children.

    

      
Property / OpValue

      
Height\(O(\log_t n)\)

      
Search / Insert / Delete\(O(\log_t n)\)

      
Split childO(t)

    

    
    
      
  • Insert (easy): put the key in a leaf that is not full (\(< 2t-1\) keys).
    • 
      
  • Insert (hard): leaf is full → must split (median goes to the parent). Parent may also be full → need a strategy so the median has somewhere to go.
    • 
      
  • Delete (easy): remove from a leaf with more than \(t-1\) keys.
    • 
      
  • Delete (hard): node is minimal (\(t-1\) keys) → borrow (shift) from a sibling or merge with a sibling before deleting.
    • 
    


    
1-pass vs 2-pass

    
Both keep the B-tree legal. The only difference is when you fix full or almost-empty nodes.

    

      
1-pass2-pass

      

        Insert
        Going down: if a node is full, split it right away. Then the leaf always has space.
        First find the leaf. If it is full, go back and split only the nodes that need it, then insert.
      

      

        Delete
        Going down: if a node has too few keys, borrow or merge now. Then delete is safe.
        First find the key. Then borrow or merge only where needed, then delete.
      

      

        Simple idea
        Fix problems on the way down.
        Find first, fix only what you must.
      

    

    
Remember: 1-pass = fix while going down. 2-pass = look first, then fix. Same big-O time.

    
A "2-3-4 tree" is a B-tree with \(t = 2\). Best answer when the exam says "data on disk" or "minimize block accesses."
