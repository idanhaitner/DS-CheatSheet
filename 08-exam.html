Chaining insert is always O(1): search/delete cost Θ(1+α) without a node pointer. Probing needs empty slots (α<1); delete probes like search, then tombstone. Rehash (doubling) costs O(n) once but O(1) amortized per insert.

8.3 Exam Strategy & Common Pitfalls

How the exam asks questions (from past papers)

  • "Best time to implement operation X on structure Y": know each structure's operation costs cold. E.g. skip-list Delete (with a pointer) is \(\Theta(1)\) expected because \(E[\text{node height}]=1\); Search without a pointer is \(\Theta(\log n)\) expected / \(O(n)\) worst.
  • Sorting best/worst fill-ins: given constraints (e.g. each element within \(n^{0.4}\) of its index), work out Insertion/Merge/Quick/Counting behavior. Insertion is O(n·shift-distance); deterministic Quick's worst is on sorted input. For linear sorts, check whether \(k=O(n)\) or \(d=o(\log n)\): see §2.3 decision guide.
  • "Best algorithm to sort \(n\) integers with \(L\) binary digits": compare Radix \(\Theta(L\cdot n)\) vs Merge \(\Theta(n\log n)\). Radix when \(L=o(\log n)\); Merge when \(L=\Omega(\log n)\).
  • Design a structure meeting target complexities: combine primitives: hash table (O(1) contains) + sorted array/AVL (order/median) + auxiliary arrays. State fields, describe each op, analyze.
  • Graph counting / proof: e.g. “number of distinct BFS trees”, "is \(e_{\min}\) in every MST?", "is edge \(e\) in some MST?": see §6.9 graph exam tips.
  • Graph reductions: super-source/sink, vertex splitting, reverse graph, reduce to MST. Always define \(G'\) and analyze build + algorithm time.

Top pitfalls to avoid

  • buildHeap is O(n), not O(n log n). Heapsort overall is O(n log n).
  • Quicksort worst = O(n²) (deterministic pivot on sorted input); average = O(n log n).
  • Counting sort is O(n+k): only linear when \(k = O(n)\); it is stable, radix depends on that. Without a bound on max key \(K\), you cannot sort in \(\Theta(N)\) with counting/radix.
  • Radix sort is \(\Theta(d(n+b))\): needs stable digit sort. With \(L\) binary digits and 1-bit passes: \(\Theta(L\cdot n)\). Compare to merge before answering "best sort".
  • Bucket sort average = O(n) but worst = O(n²) when all elements fall in one bucket.
  • Dijkstra breaks on negative edges: use Bellman-Ford.
  • BFS ≠ shortest path on weighted graphs: only unweighted hop count.
  • Spanning tree ≠ MST: MST additionally minimizes total weight; cycle/cut properties for proofs.
  • Plain BST is O(n) worst: only AVL/balanced guarantees O(log n).
  • Hash chaining: insert always O(1); search Θ(1+α) expected, O(n) worst. Probing: expected O(1) probes, worst O(n); needs α<1.
  • Master Theorem gap: doesn't apply when f differs by only a log factor.
  • Θ vs O: give Θ for a tight bound; O suffices for an upper bound unless stated.
  • State which theorem you invoke (lecture results allowed, homework results not).
