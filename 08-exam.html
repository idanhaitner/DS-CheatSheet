8.3 Exam Strategy & Common Pitfalls

How the exam asks questions

Recurring formats across past papers. Default analysis is worst-case unless the question says expected / amortized. Cite lecture theorems by name.

Paper shape

Warm-up asymptotics / recurrences → often a large always / sometimes / never block → one or two design-a-structure questions (full + cheaper partial-credit variant).

Recurrences

Equal split \(aT(n/b)+f\) → Master (need polynomial \(\epsilon\)-gap). Unequal \(T(n/3)+T(2n/3)+n\) → recursion tree → \(\Theta(n\log n)\). Otherwise iteration / substitution. Full guide: §1.2.

Always / sometimes / never

True for every input / some inputs / never. Classic traps: heap order vs BST order; Quicksort \(\Theta(n^2)\) on sorted; Bucket worst \(O(n^2)\) even without a uniform promise; AVL rotation after insert/delete at a given height.

“Best time for op X on structure Y”

Fill \(\Theta(\ldots)\) per ADT cold. Skip-list Delete with a pointer is \(\Theta(1)\) expected (\(E[\text{height}]=1\)); Search without one is \(\Theta(\log n)\) expected / \(O(n)\) worst. Hash insert \(O(1)\); search \(\Theta(1+\alpha)\) expected.

Hash table claims

AVL-per-bucket improves per-cell worst search to \(O(\log n)\), not global Successor. Probing: distinguish \(\alpha\) vs \(\alpha'\) (tombstones). Rehash / deamortized rehash appears as fill-ins or design.

Sorting fill-ins

Given constraints on \(A\) (e.g. each key within \(n^{0.4}\) of its index), fill best/worst for Insertion / Merge / Quick / Counting. Insertion \(\approx n\cdot\)shift; deterministic Quick worst on sorted. Linear sorts need \(k=O(n)\) or \(d=o(\log n)\). See §2.3.

\(L\)-bit integers

“Best worst-case sort for \(n\) numbers with \(L\) binary digits”: Radix \(\Theta(L\cdot n)\) vs Merge \(\Theta(n\log n)\). Radix when \(L=o(\log n)\); Merge when \(L=\Omega(\log n)\). Mixed ranges (most keys poly(\(n\)), few outliers): split → linear + comparison → merge.

Design a structure

Write fields first, then each op in 3–5 lines, then the bound. Combine: hash (membership) + AVL / skip / heap (order, DeleteMax); OST + size for Rank/Select; dual heaps or sorted halves when Median / SelectN must be \(O(1)\). Hard ops (Select for small \(i\), SumValues, GreaterPrice) carry the points.

Order statistics (Select / Rank)

Static one-shot → Rand-Select / MoM. Dynamic, worst-case → OST. Dynamic, expected OK → IndexableSkipList + width. Never store absolute index; Delete with width is expected \(O(\log n)\), not \(O(1)\). Toolkit: §2.4.

Partial credit variants

Full solution often needs a harder bound (e.g. Insert \(O(1)\) amortized, Init \(O(N)\), Select \(O(1)\) for \(i\le\log n\)). Weaker bound (e.g. Insert \(O(\log n)\)) still scores — state which variant you solve.

B-trees

Height \(h\le\log_t\frac{n+1}{2}\). Know 1-pass (split full child before descending) vs 2-pass (find leaf; split from lowest non-full ancestor down). Animate: §3.7.

Amortized & deamortization

Growing binary counter: \(f(k)=2k\) vs \(k+1\); token/accounting proofs; Insert amortized \(O(1)\) via periodic rebuilds. Deamortization: spread rehash (\(2m\) micro-steps over \(m/2\) inserts → ≤4/insert) so every step is \(O(1)\) worst-case. See §7.3.

Graphs

Edge membership tests (some / every MST); Kruskal and Prim; reductions. See §6.5.

Union-Find

Fill Union/Find with union-by-rank + path compression. When ranks are equal, know the lecture tie-break (which root becomes parent). Connected components = MakeSet all + Union edges.

Compression

LZ78: shortest / longest string with a fixed number of blocks. Huffman: algorithm output is optimal for that run; in general optimal prefix trees, the two rarest need not be siblings. Huffman · LZ.

Top pitfalls

  • buildHeap is \(O(n)\), not \(O(n\log n)\).
  • Heap ≠ BST order — left/right key ranges not required.
  • Quicksort worst \(O(n^2)\) on sorted (deterministic pivot).
  • Counting linear only if \(k=O(n)\); it is stable.
  • Radix \(\Theta(d(n+b))\); needs a stable digit sort.
  • Bucket avg \(O(n)\), worst \(O(n^2)\) (one bucket).
  • Dijkstra fails on negative edges → Bellman-Ford.
  • BFS = shortest hops only (unweighted).
  • Spanning tree ≠ MST — MST also minimizes weight.
  • Plain BST worst \(O(n)\); AVL for \(O(\log n)\).
  • Hash: insert \(O(1)\); search expected \(O(1+\alpha)\), worst \(O(n)\).
  • AVL-per-bucket ≠ global Successor in \(O(\log n)\).
  • Huffman is optimal for its algo; rarest need not be siblings in every optimal tree.
  • Master Theorem fails without a polynomial \(\epsilon\)-gap (e.g. \(f=n/\log n\)).
  • Prefer Θ when tight; name the theorem you use.
